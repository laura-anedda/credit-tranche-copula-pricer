# Credit-tranche Copula Pricer

`credit-copula` is a modular Python library for synthetic credit portfolio tranche valuation under a one-factor Gaussian copula framework.

It implements credit curve construction, portfolio loss modelling, expected tranche loss calculation, protection and premium leg valuation, base correlation calibration, numerical diagnostics, and static reporting.

The codebase is organized around explicit modelling assumptions, date-based cash-flow conventions, deterministic numerical methods, and reproducible example inputs.



## Features

- **Dependence model**: one-factor Gaussian copula (Li, 2000), with
  each obligor's default time driven by a latent asset value decomposed
  into a systemic factor and an idiosyncratic factor.
- **Market conventions**: ACT/360 premium accrual, ACT/365F and 30/360
  day counts, Modified Following business-day adjustment, and the
  standard backward-generated quarterly CDS/tranche schedule
  (`credit_copula.conventions`), wired into both the CDS hazard-rate
  bootstrap and tranche premium leg valuation.
- **Accrued premium on default**: included in both the CDS bootstrap
  and tranche premium leg under the standard market midpoint
  approximation, not omitted.
- **Realistic quote representation**: `credit_copula.quotes.TrancheQuote`
  distinguishes par-running-spread quotes from the fixed-coupon-plus-
  upfront convention used for equity-tranche-style instruments.
- **Portfolio loss distribution**: the semi-analytic recursive
  convolution algorithm of Andersen, Sidenius & Basu (2003), exact up to
  loss discretization granularity, for an arbitrary heterogeneous
  portfolio of notionals, recoveries, and correlations; plus the
  closed-form Large Homogeneous Portfolio (Vasicek) limit as a fast
  cross-check.
- **Tranche valuation**: risk-neutral protection and premium leg present
  values, risky PV01, par spread, and fair upfront, derived from the
  expected tranche loss (ETL) term structure.
- **Base correlation**: bootstrapping of the market-standard base
  correlation skew (McGinty et al., 2004) both from direct base-tranche
  quotes and from the standard (non-base) tranche structure via the
  sequential stripping algorithm used in practice.
- **Calibration diagnostics**: market-vs-model residuals, RMSE/max
  error, root-finder convergence diagnostics (iterations, residual,
  convergence flag), no-arbitrage checks (expected-loss monotonicity,
  base-tranche convexity), base correlation skew shape and
  curvature/stability checks, loss-discretization-error and
  quadrature-convergence diagnostics, and aggregated structured
  warnings (`credit_copula.diagnostics.CalibrationDiagnostics`).
- **Standalone HTML dashboard**: a self-contained analytics report
  (Market Inputs, Model Parameters, Calibration, Pricing Results, Risk
  Measures, Diagnostics, Visualizations) generated by a single Python
  script and opened directly in a browser — no application server.

## Repository structure

```
src/credit_copula/
    market_data.py        Discount curves (flat-forward interpolation), CDS survival curve bootstrap
    conventions.py          Day count, business-day adjustment, CDS premium schedule construction
    numerics.py             Gauss-Hermite quadrature, Brent root-finding (+ diagnostics)
    copula.py                One-factor Gaussian copula conditional default probability
    portfolio.py             Obligor/portfolio data structures, ASB recursion, LHP closed form
    tranche.py                Tranche payoff, expected tranche loss, leg valuation, accrued-on-default
    quotes.py                 TrancheQuote: par-spread vs. upfront quote representation
    base_correlation.py       Base correlation curve bootstrap (direct and standard-tranche stripping)
    pricer.py                 CDOPricer: end-to-end orchestration
    diagnostics.py            Calibration accuracy, no-arbitrage, and stability diagnostics
    reporting.py              Pure pandas reporting layer (tables for the dashboard/notebooks)
dashboard/
    illustrations.py          Monte Carlo sampling of the copula latent-variable structure (visualization only)
    charts.py                  Plotly figure builders (pure functions of reporting tables)
    layout.py                  Static HTML page assembly (CSS, sections, tables, badges)
    generate_dashboard.py      CLI entry point: builds the standalone HTML dashboard
reports/
    credit_tranche_dashboard.html   Generated dashboard output (not version-controlled)
examples/
    index_tranche_example.py  Worked example: pricing a CDX.IG-style tranche structure
tests/                        pytest suite (200+ tests across all modules)
```

Each `credit_copula` module is independently testable and free of
cross-cutting side effects; `pricer.py` is the only module in the core
library that composes the others into a single workflow.
`diagnostics.py` and `reporting.py` keep the core library free of any
UI dependency while still providing the tabular output and consistency
checks the dashboard (or a notebook, or a batch report) consumes
directly. The `dashboard/` package depends on `credit_copula` plus
Plotly only — no Streamlit, no web framework, no server process.

python3 -m venv .venv && source .venv/bin/activate
pip install -e ".[dev]"

`pyproject.toml`; there is no separate `requirements.txt` — the
project is packaged via `pyproject.toml` exclusively).


## Quick start

```python
import numpy as np
from credit_copula.market_data import DiscountCurve, bootstrap_hazard_rates
from credit_copula.portfolio import CreditPortfolio, Obligor
from credit_copula.pricer import CDOPricer
from credit_copula.tranche import Tranche

discount_curve = DiscountCurve(
    tenors=np.array([1, 3, 5, 7, 10]), zero_rates=np.full(5, 0.03)
)
credit_curve = bootstrap_hazard_rates(
    cds_tenors=np.array([1, 3, 5, 7, 10]),
    cds_spreads=np.array([0.004, 0.0065, 0.009, 0.0105, 0.012]),
    recovery_rate=0.40,
    discount_curve=discount_curve,
)

portfolio = CreditPortfolio(
    obligors=tuple(
        Obligor(f"Name_{i}", notional=1.0, recovery_rate=0.40,
                correlation=0.25, credit_curve=credit_curve)
        for i in range(125)
    )
)

pricer = CDOPricer(portfolio=portfolio, discount_curve=discount_curve, loss_unit=0.25)
tranche = Tranche(attachment=0.03 * 125, detachment=0.07 * 125)  # 3%-7%
result = pricer.price_tranche(tranche, maturity=5.0)

print(f"Par spread: {result.par_spread * 1e4:.1f} bps")
print(f"Risky PV01: {result.risky_pv01:.3f}")
```

Pass `valuation_date=date(2025, 3, 20)` to `price_tranche` to use the
full calendar-based premium schedule (real CDS roll dates, ACT/360
accrual, Modified Following payment dates) instead of the default
evenly-spaced, calendar-free approximation.

See [`examples/index_tranche_example.py`](examples/index_tranche_example.py)
for a complete worked example, including base correlation bootstrapping
across the standard CDX.IG tranche structure. The example uses
synthetic market data constructed for reproducible demonstration.

## Generating the dashboard

The dashboard is a single static HTML file, generated by a Python
script and opened directly in a browser — there is no application
server to run.

```bash
pip install -e ".[dashboard]"
python dashboard/generate_dashboard.py --open
```

This builds a reference portfolio and prices the standard CDX.IG-style
tranche structure, bootstraps base correlation, runs the full
calibration/diagnostics pipeline, and writes
`reports/credit_tranche_dashboard.html` (pass `--output PATH` to choose
a different location; omit `--open` to skip auto-launching a browser).
Generation takes under a minute on a laptop with the script's default
settings; see the module docstring in
[`dashboard/generate_dashboard.py`](dashboard/generate_dashboard.py)
for how to adjust portfolio size and numerical settings, and the
**Computational notes** below for the underlying cost trade-offs.

The market inputs in `generate_dashboard.py` are synthetic data
constructed for reproducible demonstration. The "market" tranche
spreads used in the dashboard's Calibration section are a fixed-seed
Gaussian perturbation of the model's own output, in the absence of a
live market data feed — this keeps the calibration residual
diagnostics non-trivial rather than tautologically zero. Replace
`CDS_SPREADS`, `TRANCHE_PILLARS_PCT`, and the other configuration
constants at the top of `generate_dashboard.py` with desk-sourced
market data for live use.

## Computational notes

- The Andersen-Sidenius-Basu recursion is `O(N * L)` per systemic-factor
  quadrature node, where `N` is the number of obligors and `L` the
  number of loss buckets (`total portfolio LGD / loss_unit`). Choosing
  `loss_unit` involves a direct trade-off between discretization bias
  (quantified by `credit_copula.diagnostics.estimate_loss_discretization_error`)
  and computational cost.
- Base correlation bootstrapping repeatedly reprices a base tranche
  inside a root-finder, and is therefore the most computationally
  intensive operation in the package; coarser `loss_unit` and
  `n_quadrature_points` settings are reasonable during development and
  should be tightened for final production pricing. The dashboard uses
  deliberately reduced settings (see its module docstring) to keep
  report generation fast.
- Gauss-Hermite quadrature (32 nodes by default) integrates the
  conditional loss distribution over the systemic factor to
  double-precision accuracy across the correlation ranges typically
  observed in index tranche markets (0.1-0.6); very high correlations
  approaching 1 may require additional nodes.
  `credit_copula.diagnostics.assess_quadrature_convergence` checks this
  directly by comparing model output across node counts.
